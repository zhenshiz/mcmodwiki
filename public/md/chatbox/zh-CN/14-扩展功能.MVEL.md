<h2 id="一预定义变量" data-toc-id="一预定义变量"><strong>一、预定义方法和变量</strong></h2>
<p>
  从1.1.4版本开始，ChatBox的MVEL解析支持自定义变量（包括自定义函数，在MVEL中，函数也是变量，作用范围是客户端/服务端全局）。你在对话开始时声明的变量，可以在对话过程中视情况修改，然后在文本中以<code>&lt;&lt;MVEL代码&gt;&gt;</code>的形式使用。ChatBox的MVEL解析不会主动删除自定义的变量，除非你手动删除一个自定义的变量，否则声明的变量在后续的MVEL解析时可见。
</p>
<p>以下是一些操作自定义变量的方法：</p>
<h4 id="hasitemtarget-player-itemid-string-boolean"
  data-toc-id="hasitemtarget-player-itemid-string-boolean">
  <code>hasVar(变量名: String): boolean</code></h4>
<p>判断该名称的变量是否存在，变量名必须加引号。</p>
<h4 id="hasvar变量名-string-boolean" data-toc-id="hasvar变量名-string-boolean">
  <code>setVar(变量名: String, 变量值: Object): Object</code></h4>
<p>
  添加一个变量或为变量赋值，变量名必须加引号。通常情况下你不需要用到这个方法，因为你可以直接写<code>a=10</code>，这样就完成了变量的声明和赋值。
</p>
<h4 id="setvarifnodef变量名-string-boolean"
  data-toc-id="setvarifnodef变量名-string-boolean">
  <code>setVarIfNoDef(变量名: String, 变量值: Object): Object</code></h4>
<p>
  当且仅当变量未被声明，才声明该变量，变量名必须加引号。相当于写<code>if (!hasVar('a')) {setVar('a', 10)}</code>。
</p>
<h4 id="removevar变量名-string-object" data-toc-id="removevar变量名-string-object">
  <code>removeVar(变量名: String): Object</code></h4>
<p>删除该名称的变量，变量名必须加引号。</p>
<p></p>

<h2 id="二动态方法" data-toc-id="二动态方法"><strong>三、动态方法</strong></h2>
<p>
  动态方法<s><span data-type="hidden-text">可以使用kubejs在启动脚本中用ChatBoxUtil.addMvelMethod注册（由于kubejs的bug，实际上不行），</span></s>可以在
  MVEL 中像函数一样调用。</p>
<div type="info" data-type="admonition" data-admo-type="info">
  <div data-type="admonition-title">INFO</div>
  <div data-type="admonition-content">
    <p>以下列出的方法并不是Minecraft代码里面有的方法，而是我给ChatBox的MVEL解析导入的静态方法。</p>
    <p>在通常情况下，方法应该这样使用：<code>hasItem(player,'diamond')</code></p>
    <p>
      但是我觉得静态方法用起来实在是不直观，所以ChatBox会在解析MVEL时把<code>player.hasItem('diamond')</code>改写成上面的代码<s><span data-type="hidden-text">（实际上会改写成更通用的形式，这个你不需要了解）</span></s>，就好像<code>Player</code>这个类有<code>hasItem</code>方法一样。
    </p>
    <p>当然，两种写法都是可以运行的，你喜欢怎样写都行。</p>
  </div>
</div>
<h4 id="hasitemtarget-player-itemid-string-boolean"
  data-toc-id="hasitemtarget-player-itemid-string-boolean">
  <code>hasItem(target: Player, itemId: String): boolean</code></h4>
<p>判断玩家是否拥有至少一个指定物品。</p>
<pre isclosed="false"><code>player.hasItem('diamond')   // 玩家是否有钻石</code></pre>
<h4 id="getitemcounttarget-player-itemid-string-int"
  data-toc-id="getitemcounttarget-player-itemid-string-int">
  <code>getItemCount(target: Player, itemId: String): int</code></h4>
<p>获取玩家拥有的指定物品总数（包括背包、盔甲栏、副手）。</p>
<pre isclosed="false"><code>player.getItemCount('minecraft:apple')</code></pre>
<h4 id="getitembyslottarget-livingentity-slot-string-itemstack"
  data-toc-id="getitembyslottarget-livingentity-slot-string-itemstack">
  <code>getItemBySlot(target: LivingEntity, slot: String/int): ItemStack</code>
</h4>
<p>若slot为字符串，获取生物指定装备槽位的物品。槽位名称支持：</p>
<ul>
  <li>
    <p><code>head</code> / <code>helmet</code></p>
  </li>
  <li>
    <p><code>chest</code> / <code>chestplate</code></p>
  </li>
  <li>
    <p><code>legs</code> / <code>leggings</code></p>
  </li>
  <li>
    <p><code>feet</code> / <code>boots</code></p>
  </li>
</ul>
<p>若slot为数字，获取玩家背包中指定索引的物品（索引 0~35 为主背包，36~39 为盔甲，40 为副手）。</p><pre isclosed="false"><code>target.getItemBySlot('head')   // 获取目标（若有）头盔槽物品
player.getItemBySlot(0)        // 获取玩家背包第一个物品</code></pre>
<h4 id="getitembyslottarget-player-slotindex-int-itemstack"
  data-toc-id="getitembyslottarget-player-slotindex-int-itemstack">
  <code>getScore(target: Player, objective: String): int</code></h4>
<p>获取玩家在指定计分板目标上的分数（客户端只能获取到展示的计分板的分数）。</p>
<pre isclosed="false"><code>player.getScore('example')</code></pre>
<h4 id="hastagtarget-entity-tag-string-boolean"
  data-toc-id="hastagtarget-entity-tag-string-boolean">
  <code>hasTag(target: Entity, tag: String): boolean</code></h4>
<p>判断实体是否拥有指定的标签（客户端一般情况获取不到标签，但若把该实体添加到chatbox的对话实体列表中就可以）。</p>
<pre isclosed="false"><code>player.hasTag('tag')</code></pre>
<h4 id="hasadvancementtarget-entity-tag-string-boolean"
  data-toc-id="hasadvancementtarget-entity-tag-string-boolean">
  <code>hasAdvancement(target: Player, 进度名: String): boolean</code></h4>
<p>判断玩家是否达成指定的进度。在服务端，目标可以是任意玩家，客户端只能是玩家本人，因为无法获取其他玩家的数据。</p>
<pre isclosed="false"><code>player.hasAdvancement('minecraft:story/mine_diamond')    //玩家是否取得了获得钻石进度</code></pre>
<h4 id="telltarget-player-message-string-actionbar-boolean-boolean"
  data-toc-id="telltarget-player-message-string-actionbar-boolean-boolean">
  <code>tell(target: Player, message: String, actionBar?: boolean)</code></h4>
<p>向玩家发送消息。第二个参数为 <code>true</code> 时消息显示在动作栏。</p><pre isclosed="false"><code>player.tell('你好！')           //显示在聊天框
player.tell('Hello!', true)    //显示在动作栏</code></pre>
<h4 id="telltarget-player-message-string-actionbar-boolean"
  data-toc-id="telltarget-player-message-string-actionbar-boolean">
  <code>getEnchantLevel(item: ItemStack, id: String): int</code></h4>
<p>获取物品指定附魔的等级。</p>
<pre isclosed="false"><code>player.mainHandItem.getEnchantLevel('sharpness') &gt; 0  //判断玩家主手物品是否有锋利附魔</code></pre>
<h4 id="teltarget-player-message-string-actionbar-boolean"
  data-toc-id="teltarget-player-message-string-actionbar-boolean">
  <code>enchant(item: ItemStack, id: String, level: int)</code></h4>
<p>给物品添加指定等级的附魔。需要在服务端执行才有效。</p>
<pre isclosed="false"><code>player.offHandItem.enchant('looting', 114)  //给玩家副手物品添加114级的抢夺附魔</code></pre>
<h2 id="五表达式示例" data-toc-id="五表达式示例"><strong>四、表达式示例</strong></h2>
<p>以下是一些综合示例，展示如何在 MVEL 中使用上述变量、方法和属性：</p>
<pre isclosed="false"><code>// 判断玩家是否手持钻石剑且等级大于等于10
player.mainHandItem.id == 'minecraft:diamond_sword' &amp;&amp; player.experienceLevel &gt;= 10

// 获取第一个目标的名称和健康值
target.name + ' health: ' + target.health

// 向所有目标发送动作栏消息
for (t : targets) {t.tell('Hello ' + t.name, true)}

// 随机选择一个对话目标实体
targets[random.nextInt(targets.size())]

// 判断玩家是否有“vip”标签或者拥有至少5个金苹果
player.hasTag('vip') || player.getItemCount('golden_apple') &gt;= 5</code></pre>
