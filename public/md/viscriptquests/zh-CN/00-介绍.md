<h1 id="i" data-toc-id="i">ViScriptQuests</h1>
<p>这是一款偏向RPG的任务系统，允许整合包或地图作者通过可视化的蓝图节点连线，编排带有条件分支、动态触发和复杂时空逻辑的沉浸式 RPG 剧情任务。</p>
<h1 id="与主" data-toc-id="与主">与FTBQ的相比有啥优势</h1>
<ul>
  <li>
    <p><strong>打破“清单疲劳”与剧透 (沉浸感控制)</strong></p>
    <ul>
      <li>
        <p><strong>FTBQ 痛点：</strong>
          玩家一打开任务书，庞大且复杂的任务网全部展示在眼前。这很容易让玩家产生“上班打卡”的疲劳感，且后期的终极目标（如击杀末影龙、制作终极太阳能）在第一天就被剧透。
        </p>
      </li>
      <li>
        <p><strong>VSQ 优势：</strong> 任务是“动态展开”的。玩家可能在到达特定坐标、与特定 NPC
          交互、或解锁某个前置事件后，才会在屏幕上弹出新任务。未知感更强，游戏节奏由剧情推进而非 UI 驱动。</p>
      </li>
    </ul>
  </li>
  <li>
    <p>支持更加复杂的场景</p>
    <ul>
      <li>
        <p>
          因为依赖蓝图系统，所以和FTBQ的节点图相比最大的特点是有一套成熟的变量系统，你可以利用变量系统给节点之间添加更加复杂的逻辑，对多分支的情景也会更加的友好。比如你可以在蓝图中定义一个好感度（int），在任务的进度中根据玩家做的任务或者行动去修改这个好感度来使玩家后续接到的任务或者结局来做判定。
        </p>
      </li>
      <li>
        <p>
          支持动态的目标(?)和奖励，比如说你可以根据你当前游戏的难度来动态设置任务的奖励，或者说给一个小任务设置可选目标，如果做的话会有额外的奖励。
        </p>
      </li>
    </ul>
  </li>
</ul>
